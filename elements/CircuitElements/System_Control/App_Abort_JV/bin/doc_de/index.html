BESCHREIBUNG:

T seinem Element ermöglicht es Ihnen , Befehle im auszuführen Linux - Terminal oder Windows CMD und erhält das Betriebssystem Antwort nach dem Befehl.

Dieses Element gibt , wenn ein Fehler auftritt , und ermöglicht eine Option für das Debuggen zu aktivieren, wenn diese Option ist TRUE das Element startet ein Popup-Fenster zu zeigen , was mit der Ausführung des Befehls geschieht.

BASIC Beispiel:

ELEMENT EINGÄNGE:

1, " Enable_Element " (Boolean):

Wenn Wahr den Befehl des "Befehlseingabe" in der Windows - CMD oder führt Linux - TERMINAL, wenn falsch nichts laufen.

 2, " Command_Input " (String):

Dieser Eingang ist der Befehl, den Sie in der Windows - CMD oder ausführen wollen Linux TERMINAL, müssen die Befehle mit dem Betriebssystem kompatibel sein.

 3, " Enable_Debug_Window " (Boolean):

Wenn Wahr  startet das Element eine Pop-up - Fenster zu zeigen , was mit der Ausführung des Befehls geschieht. Wenn FALSE nicht das Pop-up - Fenster zeigen.

ELEMENT AUSGÄNGE:

1, " System_Out " (String):

Dieser Ausgang ist die Antwort, die von CMD oder TERMINAL vom ausgeführten Befehl kam. Je nach Befehl und Systembetriebs diesen Ausgang kann nichts zurück.

Mit dem Beispiel, wenn Sie in Windows CMD führen Sie den Befehl " msg * \" Dies ist ein Windows - Popup-Nachricht \ " " Es ist nicht alles zurück, zeigt nur ein Pop-up - Fenster , das sagt : " Dies ist ein Windows - Pop-up Nachricht " , aber wenn Sie den Befehl ausführt" ipconfig "es gibt die entsprechenden IP - Konfigurationsdaten.

Für weitere Informationen über Betriebssystembefehle und es ergibt bezieht sich auf Ihr Betriebssystem helfen.

2, " ERROR_OUT " (Boolean):

Dieser Indikator zeigt TRUE , wenn Fehler bei der Ausführung des Befehls wurde eingegeben und FALSE , wenn kein Fehler.
